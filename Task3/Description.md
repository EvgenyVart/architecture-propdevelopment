3. Требования к внешним интеграциям
Безопасность:
      1 Идентификация:
            - Ограничение подключения по региону (но может быть будет не удобно при отпусках)
      2 Аутентификация:
           - Mutual TLS (mTLS) для всех API-вызовов
           - JWT-токены с ограниченным временем жизни (max 5 мин)
           - Обязательная верификация цифровой подписи запросов
      3 Авторизация:
            - Ролевая модель доступа (RBAC)
            - Проверка прав доступа к конкретному ЖК/объекту
            - Ограничение 1 запрос/секунду на endpoint
      4 Протоколы вынесены в отдельный файл ./ПротоколыВзаимодействия.png
      5 Организация взаимодействия:
            - Асинхронная коммуникация через шину событий (Kafka)
            - Кеширование результатов распознавания (Redis, TTL=24ч)
            - Обязательная анонимизация биометрических данных (хэширование лиц)
Дополнительные требования:
   Аудит:
        - Логирование всех событий доступа в SIEM
        - Хранение RAW-данных не более 72 часов
   Отказоустойчивость:
        - Circuit Breaker для всех внешних вызовов
        - Фоллбэк на ручное управление при сбоях
   Соответствие:
        - 152-ФЗ для персональных данных
        - ГОСТ Р 57580 (биометрия)
        - PCI DSS для платежных операций
